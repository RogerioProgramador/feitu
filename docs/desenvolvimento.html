Desenvolvimento local

Como configurar o ambiente, rodar o projeto, executar testes e adicionar novas features.

Pré-requisitos

FerramentaVersão mínimaVerificar
Java21java -version
Maven3.9mvn -version
Node.js20node -v
npm10npm -v
ℹ️

O projeto foi desenvolvido com Java 25 e Mockito em modo experimental (-Dnet.bytebuddy.experimental=true já configurado no pom.xml). Funciona normalmente com Java 21+.

Setup inicial

bash 
git clone git@github.com:RogerioProgramador/feitu.git
cd feitu
cd frontend && npm install && cd ..

Rodando em desenvolvimento

Você precisa de dois terminais abertos simultaneamente:

Terminal 1 — Backend

Backend — porta 8080 
mvn spring-boot:run -P backend-only -Dspring-boot.run.profiles=dev

O Spring Boot sobe na porta 8080. O H2 usa banco em arquivo ./data/feitu.mv.db (criado automaticamente). Console H2: http://localhost:8080/h2-console com JDBC URL jdbc:h2:file:./data/feitu.

Terminal 2 — Frontend

Frontend — porta 5173 
cd frontend
npm run dev

O Vite sobe na porta 5173 com hot-reload. O proxy /api → localhost:8080 está configurado em vite.config.ts.

🌐

Acesse http://localhost:5173

Testes

Backend

bash 
# Todos os testes (unitários + integração)
mvn verify -P backend-only

# Só compilar, sem testes
mvn compile -P backend-only

Os testes de integração usam H2 in-memory (perfil test) e @DirtiesContext para isolamento entre suítes.

Frontend

bash 
cd frontend
npm run typecheck   # verificação de tipos TypeScript
npm run test        # Vitest

Build do artefato único

bash 
# Compila frontend + backend, gera jar com tudo embutido
mvn package

# Rode localmente
java -jar target/feitu-0.0.1-SNAPSHOT.jar

O Maven Exec Plugin roda npm ci && npm run build em frontend/ antes de compilar o Java. O resultado em frontend/dist/ é copiado para src/main/resources/static/ e embutido no jar.

Critério de done

Uma funcionalidade está pronta quando:

  • 1mvn verify -P backend-only passa — compile + todos os testes
  • 2npm run typecheck passa — sem erros de tipo TypeScript
  • 3A funcionalidade está manualmente verificável via UI ou API

Variáveis de ambiente (desenvolvimento)

Em desenvolvimento, valores padrão são usados automaticamente via application.properties:

VariávelPadrão em dev
JWT_SECRETdev-secret-... (32+ chars)
JWT_EXPIRATION_MS86400000 (24h)
VAPID_PUBLIC_KEYvazio (push desabilitado)
VAPID_PRIVATE_KEYvazio

Para habilitar o console H2 em dev:

bash 
mvn spring-boot:run -P backend-only \
  -Dspring-boot.run.profiles=dev \
  -Dspring.h2.console.enabled=true

Adicionando uma nova feature

  1. Defina a entidade em domain/ e o repositório em repository/
  2. Implemente o serviço em service/ com testes unitários Mockito
  3. Exponha via controller em controller/ com testes MockMvc
  4. Adicione os tipos em frontend/src/types/index.ts
  5. Crie a API client em frontend/src/api/
  6. Implemente a store Pinia em frontend/src/stores/
  7. Crie os componentes/páginas em components/ ou pages/
  8. Verifique: mvn verify -P backend-only + npm run typecheck + npm run test
